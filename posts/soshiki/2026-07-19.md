# 2026-07-19（日）| 過保護にならない育成（責任を着地させる）

**12:00 ／ X診断（組織と仕事）／ CTA：プロフィールの note**

## X本文（フック＋選択肢）

```
「自分が責任持ちます」——その優しさが、育つ機会を奪っていることがある。

責任が本人に着地しそうになるたび、誰かがクッションを敷く。

そんな場面に立ち会った時、あなたに一番近いのはどれ？

A〜D、コメントへ。

A. その優しさに感心して、自分も同じように庇う
B. 違和感はあるが、場の空気を壊したくなくて黙って見ている
C. 自分には関係ないと、関わらないようにする
D. 「今回は本人に着地させましょう」と、静かに一言を挟む
```

## コメント①

```
過剰な優しさが、責任の着地点を消し続けることがある。

──────────────
A｜同じように庇う

優しさとしては自然な反応だ。ただし庇いが重なるほど、本人は「自分の責任」という実感を持つ機会そのものを失っていく。

──────────────
B｜違和感はあるが黙って見ている

場の空気を守る選択として理解できる。ただし黙って見ている間も、着地しない責任は静かに積み上がり続ける。

──────────────
C｜関わらないようにする

消耗を避ける選択だが、見て見ぬふりを続けると、着地しない責任がやがて別の形で組織全体のコストになって返ってくる。

──────────────
D｜「今回は本人に着地させましょう」と一言挟む

いちばん場を壊さずに効く一手だ。指摘ではなく提案の形にすると、場の空気を壊さずに済む。過剰な支援が機能不全の仕組みそのものになっていることを、誰かが一度は言葉にする必要がある。見極めが要るのは、言うタイミングだ。一つ間違えると、庇っていた側の面子を潰す形になりかねない。

──────────────

優しさは、着地点を作ることと矛盾しない。
```

## コメント②

```
「クッションを敷きすぎない」ための線引き：

1. 助けるのは「やり方」まで。「責任の所在」までは肩代わりしない
2. 本人が失敗した時、結果を淡々と本人に返す（罰ではなく、事実として）
3. その上で、次に向けた支援は惜しまない

優しさが機能不全の仕組みになっていないか、時々点検する。
本当の優しさは、いつか本人が自分の足で立てるようにすることだ。

──────────────

→ 優しさは、着地点を作ることと矛盾しない。「過保護にならない育成の仕組み」は、プロフィールのnoteへ。あなたが今、代わりに敷いているクッションはあるだろうか。
```
